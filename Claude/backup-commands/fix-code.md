## 执行过程

1. **读取项目文档**
   - 阅读项目的文档和代码注释，理解整体架构和关键功能。
   - 了解项目技术栈（Unity + C#），确保正确处理游戏逻辑的各个模块。

2. **深度思考**
   - 在执行计划前深入思考。创建一个解决所有要求的综合计划。
   - 使用你的待办事项工具将复杂任务分解为更小、可管理的步骤。
   - 使用 TodoWrite 工具创建和跟踪你的实施计划。
   - 从现有代码中识别要遵循的实施模式。

3. **规划计划过程**
   - 深入思考你的完成目标需要的执行步骤，使用 TodoWrite 工具将每个具体的步骤记录下来
   - 检查你的 TODO List 计划，是否遗漏了重要步骤
   - 确保每个步骤都是可执行的，并且有明确的完成标准

4. **执行计划**
   - 执行测试
   - 修复错误

5. **完善测试用例**
   - 如果该错误能通过测试用例避免，那么你应该为此编写或者修改当前已有的测试用例，以覆盖这种错误情况
   - 覆盖边界情况和错误处理
   - 确保测试用例可重复运行

6. **验证**
   - 运行每个验证命令
   - 修复任何失败
   - 重新运行直到全部通过
   - 如果本次运行验证发现了错误，重新执行 **验证过程** 这个工作过程。只有本轮验证全部一次性通过，才能进入下一个工作过程。

7. **完成**
   - 确保所有检查清单项目已完成
   - 如果为了测试添加了测试日志或者测试代码，结束任务前记得清除它们
   - 运行最终验证套件
   - 报告完成状态
