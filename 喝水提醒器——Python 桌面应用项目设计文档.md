## 1. 项目概述

### 1.1 背景与目标
“喝水啦”是一款专为 **3-8 岁儿童** 设计的移动端喝水习惯养成游戏。它摒弃复杂的功能和文字，用一只活泼可爱的虚拟伙伴陪伴孩子成长。小朋友每次喝完水后，只需点击一个色彩鲜艳的大按钮，伙伴就会获得能量、积累经验并逐渐进化成新的形象。所有的交互反馈都由动画、表情和简单音效传达，无需阅读，让孩子在“喂养伙伴”的乐趣中主动爱上喝水。

应用完全离线运行，没有任何网络功能、社交入口或第三方广告，家长可放心让孩子使用。通过内置的防刷机制和可配置的每日目标，既保护了记录的真诚性，又可根据孩子实际情况灵活调整。

目前版本中，伙伴的形象将使用 **临时占位的 2D 精灵动画** 代替最终的 3D 模型，成就系统也以 **简单的贴纸卡片** 呈现。具体进化形态名称、成就名称、水分值状态划分及所有游戏数值均由后期策划团队统一设定，当前仅以占位资源或示例名称标明功能存在。

### 1.2 核心特色
- **纯子供向界面**：所有元素均采用圆润造型、明亮色调和超大点击区域，无文字说明，仅凭图标和表情即可理解。
- **一只大按钮**：全屏唯一操作对象，点击即代表“我喝水了”，反馈直接，无需任何导航。
- **伙伴进化与反馈**：水分值和经验驱动伙伴外观变化，具体状态名称、触发阈值及进化形态列表由策划后期设定，当前使用占位动画表现。
- **占位素材灵活替换**：当前使用手绘风格 PNG 序列帧动画模拟伙伴，成就徽章为静态小卡片。未来可直接导入 Blender 模型（如 .obj 或 .glTF 格式）和精美贴纸资源，不改动核心代码。
- **家长友好**：通过隐藏式家长设置（需做简单加法验证）调整每日杯数、防刷间隔和上限，避免孩子误操作。
- **纯净单机**：无联网、无社交、无广告、无内购，永远免费。

## 2. 游戏化核心设计

### 2.1 养成伙伴（占位名待定）
每个孩子拥有一位专属伙伴，默认名为策划给定的可爱昵称（如“小水滴”可作占位）。伙伴由以下几项属性驱动，所有属性变化都会触发对应的视觉和音效反馈。

| 属性         | 作用                                               | 初始值 | 上限/衰减说明                                      |
| ------------ | -------------------------------------------------- | ------ | -------------------------------------------------- |
| 水分值       | 反映伙伴当前“口渴”程度，随时间持续下降，喝水后上升 | 可配置 | 上限由每日目标杯数决定；自然衰减公式由数值策划制定 |
| 经验值       | 积累后提升等级，每级所需经验逐渐增加               | 可配置 | 无上限                                             |
| 等级         | 经验值达到阈值后自动升级，触发进化动画和形态切换   | 1      | 最高等级由策划决定                                 |
| 连续天数     | 每天至少记录一次喝水可延续，中断归零               | 0      | 无上限                                             |
| 今日杯数     | 当天已记录的喝水次数，每日零点重置                 | 0      | 受每日最大杯数限制                                 |
| 今日目标达成 | 布尔值，用于一次性发放额外经验                     | false  | 每天重置                                           |

### 2.2 核心游戏循环
```
孩子喝完水 → 点击“喝水啦”按钮 → 防刷验证通过 → 水分值上升 + 经验增加 → 伙伴表现开心动画和音效 → 若经验足够则进化 → 伙伴外观永久变化 → 期待下一次喝水
```
整个循环没有任何惩罚机制。即使孩子忘记喝水导致水分值归零，伙伴也不会消失或惩罚，只会呈现“没精神”的表情，鼓励孩子赶快喂它。

### 2.3 游戏化元素
- **形态进化**：等级提升触发伙伴形态改变。具体进化阶段划分、各阶段形态名称与对应等级由策划确定，当前使用 **占位形态A～E** 的序列帧动画（如形态A、形态B等）。每个形态包含若干状态动画（状态名称待定，如状态1、状态2等）。
- **每日目标星星**：主界面以星星图案展示当日喝水进度，完成目标时所有星星亮起，伙伴表演欢呼动画。
- **连续天数火焰**：连续天数以燃烧的小火苗和数字展示，数字越大火苗越旺，同时喝水经验加成比例提高。
- **贴纸式成就**：完成里程碑后获得漂亮的小贴纸，可收集在贴纸册中。成就列表、解锁条件及对应贴纸设计由策划后期提供，当前使用 **占位成就1～5** 的简单贴纸图片。

### 3.2 MVVM 架构详解
```
View (Kivy Widgets)  <-- binding -->  ViewModel (Properties)  <-- invoke/notify -->  Model (业务逻辑)
```
- **Model**：纯 Python 类，不依赖 Kivy。包含 `Companion`（伙伴属性与进化规则）、`DailyTracker`（今日计数、连续天数管理）、`AntiCheat`（冷却与上限验证）、`AchievementManager`（成就解锁条件）、`PersistenceService`（JSON 读写）。
- **ViewModel**：持有 Model 实例，将所有数据转化为 `Observable` 属性（如 `NumericProperty`、`BooleanProperty`），并暴露命令方法（`drink_water_command`）。负责游戏循环调度：例如记录喝水后更新伙伴，计算按钮是否可用，判断何时触发进化动画。
- **View**：由 `.kv` 文件定义布局，绑定 ViewModel 属性。伙伴动画区域使用一个 `Image` 组件，通过 `source` 属性循环切换序列帧；按钮状态通过 `disabled` 绑定；进度星星、连续天数火焰等为定制组合组件。View 不直接访问 Model。

**数据流动示例**：孩子点击按钮 → View 调用 `viewmodel.drink_water_command()` → ViewModel 调用 Model 进行防刷验证，若通过则更新伙伴状态并保存 → ViewModel 更新自身属性（水分值、经验、按钮 disabled 等） → View 自动重新渲染。

## 4. 核心功能：喝水记录与游戏循环

### 4.1 喝水记录流程（防刷细节）
1. 孩子点击按钮，View 触发 `viewmodel.drink_water()`。
2. ViewModel 获取当前系统时间 `now` 和上次记录时间 `last_record_timestamp`。
3. 检查是否跨天：如果 `now.date() != last_record_timestamp.date()`，则重置 `today_cups` 为 0，`today_target_completed` 为 False，并更新连续天数（若上次日期是昨天则 +1，否则重置为 0）。
4. 防刷验证：
   - 计算 `elapsed = now - last_record_timestamp`（秒），与冷却时间（可配置）比较。若不足，返回“冷却中”。
   - 检查 `today_cups` 是否大于等于每日最大杯数（可配置）。若是，返回“已达上限”。
5. 验证通过后，Model 更新：
   - `today_cups += 1`
   - `hydration = min(hydration + 单杯水量增加值, 上限值)`（增加值、上限均由策划设定）
   - `exp += 单次喝水经验 * (1 + streak_bonus)`（加成规则由数值策划设定）
   - 检查目标达成：`today_cups >= daily_target` 且 `today_target_completed == False`，则额外发放目标完成经验（数值可配），并标记 `today_target_completed = True`。
   - 检查成就解锁（成就列表由策划提供）。
   - `last_record_timestamp = now`
6. ViewModel 通知 View 播放喝水音效、伙伴开心动画，并更新所有绑定属性。若经验使等级提升，触发进化动画。

### 4.2 防刷参数
- **冷却时间**：可配置（默认值占位，如5分钟），家长可在设置中调整。冷却期间按钮置灰，点击时弹出提示（内容由策划提供，如“再等等哦”）。
- **每日上限**：可配置（默认值占位，如15杯）。达到上限按钮置灰，点击时弹出提示（内容由策划提供，如“今天喝够啦”）。
- 按钮置灰时，外观保持原样（文字与图标不变），仅透明度或颜色变化，点击时弹出 Toast。

### 4.3 伙伴水分值模型（具体参数待定）
- 水分值有上限，由每日目标杯数决定。
- 喝水增加值、自然衰减速率均由数值策划配置，保证约数小时从满值衰减至低位。
- 水分值划分为多个状态区间，每个区间驱动不同的伙伴动画（状态名称、阈值、动画资源由策划提供）。当前占位使用“状态A”、“状态B”等表示。
- 无惩罚机制，伙伴不会消失，仅外观改变。

## 5. 养成系统与成长体系

### 5.1 经验与等级公式（参数待定）
- 基础经验：每次喝水获得固定经验（数值可配）。
- 目标达成奖励：当日首次完成目标时额外获得经验（数值可配）。
- 连续天数加成：不同天数段对应不同加成比例，具体分段与比例由数值策划制定。
- 升级所需经验：使用公式 `EXP_required(level)`，参数可调，确保前期升级快、后期放缓。
- 升级时水分值恢复行为（如回满或增加固定值）由策划设定。

### 5.2 形态进化（占位方案）
当前使用 **2D 序列帧动画** 表示不同进化阶段。每个阶段对应一个占位形态（名称如“形态A”、“形态B”……）。具体等级阈值、形态数量及对应资源由策划决定，开发时通过配置文件（如 `evolution.json`）加载。
- 进化触发：ViewModel 检测到等级跨越阈值后，暂停主界面，播放全屏进化动画（由黑屏渐变显示新形态，伴随和弦音效）。
- 伙伴当前形态由 `companion.level` 决定，View 根据等级范围加载对应序列帧文件夹。

### 5.3 成就系统（占位方案）
当前以简单贴纸卡片的形式展示，每个成就解锁时弹出贴纸动画。
- 成就定义由策划提供列表，至少包含ID、解锁条件、资源路径。当前使用占位成就ID（如 `achievement_1`）和占位贴纸图片。
- 成就数据存储在 `game_state.json` 的 `unlocked_achievements` 列表中。
- 成就页面展示已解锁贴纸与灰色未解锁轮廓。

## 6. 数据持久化方案

### 6.1 存储文件结构
#### `user_config.json`
包含用户配置项：每日目标杯数、冷却时间、每日最大杯数、伙伴昵称、音效开关等。具体键值由开发定义，所有数值为可配默认值。

#### `game_state.json`
包含运行时状态：水分值、经验、等级、连续天数、今日杯数、目标达成标记、最后记录时间戳、已解锁成就列表等。

#### `drink_log.json`
记录每次喝水的详细信息（时间戳、杯数），保留最近90天。

### 6.2 读写与异常处理
- 原子写入：临时文件+替换，保留备份。
- 加载：启动时读取，异常时从备份恢复或初始化默认数据。
- 保存频率：喝水后2秒防抖，进入后台/退出时立即保存。
- 跨天处理：由 ViewModel 在记录前主动调用日期检查。

## 7. 用户界面与交互设计

### 7.1 设计原则
- 超大点击区域，色彩柔和明亮，极少文字。
- 所有反馈为动画+音效+简单提示（Toast）。
- 导航直观，图标为主。

### 7.2 主屏幕布局
- 背景：可配置主题背景（占位使用彩虹渐变）。
- 伙伴动画区域：显示当前形态的序列帧。
- 水分值条：自定义样式（如云朵进度条）。
- 等级显示：星星或数字。
- 喝水按钮：圆形大按钮，固定文字“喝水啦”或图标。冷却/上限时仅置灰，点击弹出提示。
- 今日进度：星星或图标表示已完成/未完成杯数。
- 连续天数火焰：图标+数字。
- 底部导航：主页、贴纸、家长。

### 7.3 动画状态机
伙伴动画优先级：进化 > 特殊状态（如喝饱） > 各水分值状态。状态切换有过渡效果。具体动画资源与状态对应关系由配置驱动。

### 7.4 贴纸界面
- 收集册风格，已解锁彩色，未解锁灰色轮廓。

### 7.5 家长设置
- 进入需算术验证。
- 可调整：每日目标、冷却时间、每日上限、昵称、音效开关、重置数据。
- 修改即时保存并反映到主界面。
