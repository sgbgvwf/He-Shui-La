# 项目上下文提示工程

这个仓库用于指导 Claude 在项目中的行为和代码生成规范。

重要！！！请牢记 AI Agent 只是你的助手，不要让他做完所有的工作，否则可以开除你了，你也就不配拥有任何的回报。

本仓库作者已经对 AI Agent 有丰富的开发/使用经验，结论仍是：AI Agent 有着巨大局限性。

AI 在上下文信息的掌握上尤为孱弱。我们任何的努力都只是为了让 AI Agent 获得尽量正确和完整的上下文，并且还要防止它遗忘。

在你如同仓库作者那样对 AI Agent 有了足够的了解之前，不要怀疑本文中的最佳实践，要么不要使用 AI Agent，要么遵循最佳实践。

## 如何使用

1. 请将此仓库放到项目仓库的同级目录下。
2. 在 IDE 中安装 Claude Code 插件并配置 API Key。
3. 通过 Claude Code 启动 AI 助手。

## 最佳实践

### 如何提需求

你可以开始提需求了，建议以下几种需求提法：

#### 直接要求 Claude 帮你修复某个 Bug

用文字描述清楚"现状"和"期望"，这样才能给 Claude 足够的信息去理解需求。

```
修复 XX 系统的 XX Bug，表现为 XX，期望 XX
```

#### 策划式提法

注意，让 Claude 作为策划时，必须要求清晰，不能偷懒，否则 Claude 会胡乱发挥。

```
请你作为一个资深的游戏数值策划，为主线关卡奖励配置数值，现在的主线关卡都是一样的奖励。你可以从装备的配置数据中获得装备的信息，你需要作为一个资深游戏数值策划，来合理进行奖励的数值配置，你可能需要继续研究我们的装备系统，例如我们的装备有升级/合成升阶的概念，如果文档中缺失相关概念，你需要根据目前的程序实现来进行补充，以便下次我们根据文档来迅速了解我们的系统。
```

#### 程序式提法

此时，通常只需要简单明了地提出程序修改需求。为了让 Claude 准确找到相应模块，你需要明确指出一些模块的代码命名或者代码文件命名。

```
修改 PlayerController.cs 中的移动逻辑，添加冲刺功能
```

### 代码审查

需要审查某个 Unity C# 脚本时：

```
审查 [文件路径]
```

### 设计文档

需要为某个系统设计策划文档时：

```
作为资深游戏策划师，为 [系统名] 设计策划文档
```

## 已知局限

1. Claude 无法直接运行 Unity 编辑器，只能在代码层面进行分析和修改
2. Claude 对 Unity 场景文件和 Prefab 的理解有限，涉及场景结构的问题需要程序员辅助
3. 大型重构任务建议拆分为多个小步骤逐一完成
