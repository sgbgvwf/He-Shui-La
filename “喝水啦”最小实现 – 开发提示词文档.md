# “喝水啦”最小实现 – 开发提示词文档

> 本文档用于指导开发团队或 AI 助手，基于项目设计文档 v4.9 构建一个**可运行的最小可行产品（MVP）**。  
> 目标：在 1～2 周内完成核心闭环，所有美术与数值均使用占位资源，功能完整，可安装到真机体验。

---

## 1. 实现范围

本次最小实现聚焦于**核心游戏循环**，裁剪或简化非必要功能，确保应用可以启动、记录喝水、反馈伙伴状态、保存数据。

### 1.1 必须实现的功能
- 主界面显示：伙伴占位动画（简单序列帧或静态图片）、水分值进度条、经验/等级显示、今日星星进度、连续天数火焰、圆形喝水按钮。
- 点击按钮 → 防刷验证（冷却 + 每日上限） → 更新水分值、经验、等级、今日杯数、目标达成、连续天数 → 保存状态 → 刷新界面。
- 跨天重置今日杯数和目标标记，更新连续天数。
- 数据持久化：`game_state.json` 和 `user_config.json`，启动时加载，每次喝水后保存。
- 基础的家长设置入口（隐藏，需简单算术验证）：调整每日目标、冷却时间、每日上限。
- 贴纸（成就）界面：只显示占位贴纸，无需解锁逻辑（可放一个静态网格）。

### 1.2 暂不实现（后续迭代）
- 3D 模型支持（保留序列帧即可）
- 全屏进化动画（改为简单弹窗或 Toast 提示）
- 音效（用视觉反馈替代，或加载一个占位音效文件）
- 复杂状态机（伙伴仅用静态图片或预置几帧）
- 详细日志文件（只保留状态 JSON）
- 多语言、主题切换

- **语言**：Python 3.10+
- **UI 框架**：Kivy 2.2+（+ KivyMD 1.1+ 可选用，但建议先用纯 Kivy 减少依赖）

- **动画**：`kivy.animation.Animation` 或直接切换 `Image.source`（序列帧）
- **测试**：`pytest`（可选，但建议至少写核心单元测试）
- **版本控制**：Git，采用 `develop` 分支开发

#### `companion.py`

class Companion:
    def __init__(self, level=1, exp=0, hydration=100):
        self.level = level
        self.exp = exp
        self.hydration = hydration   # 0 ~ max_hydration

    def add_exp(self, amount):
        self.exp += amount
        # 检查升级（升级所需经验由公式决定）
        while self.exp >= self.exp_for_next_level():
            self.level += 1
            self.exp -= self.exp_for_next_level()  # 或重新计算
            # 升级时可恢复部分水分值（可配置）

    def exp_for_next_level(self):
        # 占位公式：level * 100
        return self.level * 100

    def max_hydration(self):
        # 由每日目标决定（例如 target_cups * 20）
        return target_cups * 20
```

class AntiCheat:
    def __init__(self, cooldown_seconds=300, max_cups_per_day=15):
        self.cooldown = cooldown_seconds
        self.max_cups = max_cups_per_day

    def can_drink(self, last_timestamp, today_cups, now):
        if today_cups >= self.max_cups:
            return False, "已达上限"
        if (now - last_timestamp).total_seconds() < self.cooldown:
            return False, "冷却中"
        return True, ""
```

### 4.2 ViewModel 层

#### `main_viewmodel.py`（继承 `kivy.event.EventDispatcher` 或使用 `kivy.properties`）
```python
from kivy.properties import NumericProperty, StringProperty, BooleanProperty, ObjectProperty

class MainViewModel:
    # 暴露给 View 的属性
    hydration = NumericProperty(100)
    level = NumericProperty(1)
    exp = NumericProperty(0)
    exp_next = NumericProperty(100)
    today_cups = NumericProperty(0)
    target_cups = NumericProperty(8)
    streak = NumericProperty(0)
    target_completed = BooleanProperty(False)
    button_disabled = BooleanProperty(False)
    partner_image = StringProperty('resources/sprites/idle.png')

    # 内部引用
    companion: Companion
    tracker: DailyTracker
    anti_cheat: AntiCheat
    config: dict

    def __init__(self):
        self.load_config()
        self.load_state()
        self.update_ui_properties()

    def drink_water(self):
        now = datetime.now()
        # 1. 跨天重置
        self.tracker.reset_if_new_day(now)
        # 2. 防刷检查
        can, msg = self.anti_cheat.can_drink(
            self.last_timestamp, self.tracker.today_cups, now
        )
        if not can:
            # 显示 Toast 提示
            return
        # 3. 更新状态
        self.tracker.today_cups += 1
        hydration_gain = 20  # 占位
        self.companion.hydration = min(
            self.companion.hydration + hydration_gain,
            self.companion.max_hydration()
        )
        exp_gain = 10 + self.tracker.streak * 2  # 加成占位
        self.companion.add_exp(exp_gain)
        # 检查目标达成
        if self.tracker.today_cups >= self.tracker.target_cups and not self.tracker.target_completed:
            self.tracker.target_completed = True
            # 额外经验奖励
            self.companion.add_exp(20)  # 占位
        # 更新连续天数（在 reset_if_new_day 已处理）
        self.last_timestamp = now
        # 更新 UI 属性
        self.update_ui_properties()
        # 保存状态
        self.save_state()
        # 触发伙伴开心动画（切换图片）
        self.partner_image = 'resources/sprites/happy.png'
        # 过一会儿恢复为 idle（可使用 Clock.schedule_once）

    def update_ui_properties(self):
        self.hydration = self.companion.hydration
        self.level = self.companion.level
        self.exp = self.companion.exp
        self.exp_next = self.companion.exp_for_next_level()
        self.today_cups = self.tracker.today_cups
        self.streak = self.tracker.streak_days
        self.target_completed = self.tracker.target_completed
        # 按钮禁用条件
        now = datetime.now()
        can, _ = self.anti_cheat.can_drink(self.last_timestamp, self.tracker.today_cups, now)
        self.button_disabled = not can
```

## 6. 占位资源准备

- **伙伴图片**：至少准备两张 PNG（`idle.png`, `happy.png`）放在 `resources/sprites/` 下。尺寸建议 512x512，纯色或简单卡通风格。
- **星星图标**：空星和实星两张图。
- **火焰图标**：一张火焰图。
- **贴纸**：5 张不同颜色的方形图片，命名 `sticker_1.png` ~ `sticker_5.png`。

这些资源可用在线工具快速生成或使用系统自带图标（如 Kivy 的 `kivy.atlas` 内置）。

## 7. 持久化与启动流程

1. `main.py` 中继承 `App`，在 `build()` 中：
   - 实例化 `MainViewModel` 并存入 `app.viewmodel`。
   - 加载配置和状态。
   - 创建 `ScreenManager` 并添加主界面、贴纸界面、设置界面。
2. 每次 `drink_water()` 后，调用 `persistence.save_game_state()` 保存状态。
3. 在 `on_stop()` 中再次保存，确保退出时数据持久化。

## 10. 开发注意事项

- 所有数值（如单次加水量、经验值、升级经验曲线）暂时硬编码，但应抽取为配置常量，方便后期调整。
- 界面布局必须适配不同屏幕尺寸，使用 `dp` 和相对大小。
- 按钮禁用时仍应可点击以显示 Toast 提示（说明原因）。可在 `on_press` 中检查 `button_disabled` 并显示 Toast（Kivy 的 `Toast` 或自定义弹窗）。
- 保持代码整洁，添加必要注释，为后续替换真实资源留出接口。
