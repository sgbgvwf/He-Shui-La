# 更新 Claude Code 工作提示文档

AI 工作流描述: $ARGUMENTS

## 背景知识

Claude Code 是一个 AI 工具，驱动它工作需要对工作及其上下文进行描述，以便 AI 能更好地理解和执行任务。

AI 工具的上下文窗口有限，容易忘记早些时候获取到的信息。

Claude Code 会从 /CLAUDE.md 文档开始加载信息。

## 你现在要做的事情

你要做的就是，为了让 Claude Code 能更好地完成工作，新增或完善或精炼工作提示文档。这个提示文档的范畴是关于 $ARGUMENTS 的。

AI 工作提示文档应当具有以下要点：

1. 尽量精简。因为 AI 工具的上下文窗口有限，过长的文本可能导致重要信息被截断或忽略，导致 AI 工作的效果不佳。
2. 为了让 AI 能获取到必要的足够信息，可以通过 "路由" 文档来指引 AI 在何目录下去读取相应的文档。这个手段可以补足 AI 工具的上下文窗口有限的缺点。
3. 为了防止 AI 忘记一些重要信息，可以在工作提示文档中强调 AI 可以使用 TodoWrite 工具来记录工作步骤，或者提示 AI 在执行过程中重新读取文档或者代码来强化记忆。

## 你现在要做的工作过程

1. **加载信息**

    - 理解所有上下文和要求。
    - 阅读已有的功能文档，避免错漏或者冗余，确保整体文档的一致性。
    - 分析需求，将需求拆解/细化为具体的编写步骤。
    - 遵循功能简介中的所有指示，并根据需要扩展研究。
    - 确保拥有完全实现该功能所需的所有上下文。
    - 根据需要进行更多网络搜索和代码库探索。

2. **深度思考**

    - 在执行计划前深入思考。创建一个解决所有要求的综合计划。
    - 文档的添加需要考虑现有功能，判断如何编写文档才是最为合理的，不要在没有充分阅读其他功能文档和目标代码的情况下就开始改文档。
    - 使用你的待办事项工具将复杂任务分解为更小、可管理的步骤。
    - 使用 TodoWrite 工具创建和跟踪你的实施计划。
    - 从现有代码/文档中识别要遵循的实施模式。

3. **规划计划过程**

    - 深入思考你的完成目标需要的执行步骤，使用 TodoWrite 工具将每个具体的步骤记录下来。
    - 检查你的 TODO List 计划，是否遗漏了重要步骤。
    - 确保每个步骤都是可执行的，并且有明确的完成标准。

4. **执行计划**

    - 执行你的计划。
    - 编辑生成的文档，确保其准确性和完整性。

5. **验证**

    - 通过当前已有的文档信息反查工程的实现，确保其一致性和正确性，避免文档比实现多余。
    - 如果本次运行验证发现了错误，重新执行 **验证过程** 这个工作过程。只有本轮验证全部一次性通过，才能进入下一个工作过程。

6. **完成**

    - 确保所有检查清单项目已完成
    - 报告完成状态
    - 再次阅读功能需求文档确保已实现所有内容

7. **参考功能需求文档**
    - 如需要可以随时再次参考功能需求文档
